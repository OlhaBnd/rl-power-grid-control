# ⚡ RL Power Grid Topology Control

> Бакалаврська дипломна робота  
> Спеціальність: 121 — Інженерія програмного забезпечення  
> Кафедра інженерії програмного забезпечення в енергетиці

## 📋 Тема

**Глибоке навчання з підкріпленням для керування топологією енергомереж при раптових втратах вузлів генерації**

## 🎯 Мета

Розробка інтелектуальної системи керування топологією енергомережі IEEE 14-bus на основі алгоритму PPO (Proximal Policy Optimization), що здатна запобігати блекаутам при раптових відключеннях вузлів генерації.

---

## 🏗️ Архітектура системи

### Observation Space (114 параметрів)
| Параметр | Опис | Розмір |
|----------|------|--------|
| `rho` | Завантаженість ліній (% від максимуму) | 20 |
| `p_or` | Активна потужність на початку лінії | 20 |
| `gen_p` | Генерація активної потужності | 6 |
| `load_p` | Споживання навантажень | 11 |
| `topo_vect` | Топологічний вектор вузлів | 57 |

### Action Space (101 дія)
- **1** дія — нічого не робити (do nothing)
- **40** дій — підключити/відключити кожну з 20 ліній
- **60** дій — перемикання шин у підстанціях

### Reward Function

$$R(t) = \alpha \cdot S(t) - \beta \cdot O(t) - \gamma \cdot L(t) + \delta \cdot B(t)$$

| Компонент | Назва | Коефіцієнт | Опис |
|-----------|-------|------------|------|
| S(t) | Стабільність | α = 1.0 | +1 за кожен крок без аварії |
| O(t) | Штраф перевантаження | β = 2.0 | -2 × сума перевантажень > 90% |
| L(t) | Штраф блекауту | γ = 100.0 | -100 при блекауті |
| B(t) | Бонус з'єднаності | δ = 0.5 | +0.5 × частка активних ліній |

## 📈 Гіперпараметри PPO

| Параметр | Значення | Призначення |
|----------|----------|-------------|
| learning_rate | 3×10⁻⁴ | Швидкість навчання |
| n_steps | 2048 | Кроків до оновлення |
| batch_size | 64 | Розмір міні-батчу |
| n_epochs | 10 | Епох на оновлення |
| ent_coef | 0.01 | Коефіцієнт ентропії |
| clip_range | 0.2 | Діапазон кліпування |
| gamma | 0.99 | Дисконт майбутніх нагород |
